# Stellar Suite Banner Guidelines

## Design Specifications

These guidelines define the visual identity for the Stellar Suite VS Code marketplace listing and related promotional materials.

### 1. Typography
- **Primary Font**: System fonts (San Francisco on macOS, Segoe UI on Windows, Roboto on Android, Inter for web).
- **Weight**: 
  - Title: Bold (700)
  - Subtitle: Medium (500)
  - Features: Semi-Bold (600)
- **Tracking**: Tight letter-spacing for titles (-1px to -2px), loose for feature lists (1.5px to 2px).

### 2. Color Palette
- **Background Gradient**:
  - Dark Blue: `#0F0F1A`
  - Deep Navy: `#1A1A2E`
- **Text**:
  - Primary (Title): `#FFFFFF`
  - Secondary (Subtitle): `#A1A1AA` (Zinc 400)
  - Accent (Features): `#60A5FA` (Blue 400)
- **Glow & Accents**:
  - Blue: `#3B82F6`
  - Purple: `#8B5CF6`

### 3. Key Messaging (Extension Value)
The banner must clearly communicate the core functionality:
- **Title**: Stellar Suite
- **Subtitle**: Build, Deploy, and Simulate Smart Contracts in VS Code
- **Highlights**: INTERACTIVE CLI • ONE-CLICK DEPLOYS • RPC MANAGEMENT

### 4. Marketplace Best Practices
- **Visibility**: Use high contrast (dark background with white/neon text) to stand out on both light and dark VS Code marketplace themes.
- **Simplicity**: Do not clutter the banner with UI screenshots. Screenshots belong in the README body. The banner should focus on branding and value proposition.
- **Aspect Ratios**:
  - **Marketplace Header**: 1024x256 (4:1 ratio)
  - **Social / OpenGraph**: 1280x640 (2:1 ratio)
  - **Sidebar / Minimal**: 300x250

### 5. Brand Assets
The extension uses the standard Stellar-inspired colors to indicate its ecosystem connection while maintaining a distinct "developer tooling" aesthetic. The glowing node/network design abstractly represents smart contracts deployed on the Stellar network.
